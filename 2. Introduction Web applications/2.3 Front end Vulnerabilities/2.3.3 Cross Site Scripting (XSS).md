`HTML Injecction` vulnerability can be used to perform [Cross-Site Scripting (XSS)](https://owasp.org/www-community/attacks/xss/) attacks by injecting `JavaScript` code to be executed on the client-side. Once we can execute code on victims machine, their account can be taken over or even their machine. It is similar to `HTML Injection` but instead of `HTML` it passes `JavaScript` code.

There are three main types of `XSS`:

|Type|Description|
|---|---|
|`Reflected XSS`|Occurs when user input is displayed on the page after processing (e.g., search result or error message).|
|`Stored XSS`|Occurs when user input is stored in the back end database and then displayed upon retrieval (e.g., posts or comments).|
|`DOM XSS`|Occurs when user input is directly shown in the browser and is written to an `HTML` DOM object (e.g., vulnerable username or page title).|
Payload to trigger XSS

```
#"><img src=/ onerror=alert(document.cookie)>
```
